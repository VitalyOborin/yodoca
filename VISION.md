# Yodoca Vision

Yodoca - это ИИ агент, который может развивать сам себя.
Первоначально это был эксперимент - создать приложение, которое может себя дописывать. Это должен был быть простой ИИ агент с инструментами для собственного развития. Ты ему говоришь "давай общаться в Telegram" и он дописывает необходимый функционал, от тебя требуется лишь дать Telegram Bot Secret для начала общения. И это сработало.

Это приложение для таких же любопытных инженеров-энтузиастов как и я сам.

## Приоритеты
* Стабильность работы
* Простота для пользователя
* Возможности расширения

## Цели развития приложения
### Суверенитет данных
Приложение должно полностью работать локально на компьютере пользователя и даже без доступа в интернет. Современные модели развиваются сумасшедшими темпами, сегодня вполне реально запустить локально на игровой видеокарте маленькую модель, которой будет достаточно для повседневных задач. Вспомогательные инструменты вроде embedding и подавно. Локальная модель может быть развернута в LM Studio и других подобных приложениях. 
Но использование внешних провайдеров для инференса - это опция и для многих она может быть более удобной. За $20-30 в месяц вы получается гораздо более продвинутые современные модели, более быстрые ответы.

### Абсолютная расширяемость
Изначально было решено, что в приложении всё является расширением. Стабильное ядро содержит только агент оркестратор и базовые инфраструктурные функции (event bus, интерфейсы расширений, утилиты и так далее). Всё остальное - extensions. Приложение или пользователь могут их менять, клонировать, дописывать новые, что угодно. Более того, ИИ агент сам может их себе дописывать по запросу.

### Event Driven
Все взаимодействия через события. Это дает хороший decoupling, независимость частей приложения.

### Простота
Приложение должно быть простым и легким. В базовой версии приложение должно обеспечивать минимальный функционал, достаточный для запуска и выполнения простых задач и сценариев. Всё остальное - через новые extensions, skills, агентов. Каждый пользователь может создать из этой основы что угодно своё. Минимум зависимостей, никаких сложных инфраструктурных вещей вроде Docker, PostgreSQL, Kafka и так далее. Достаточно пакетов Python + SQLite для большинства задач.
